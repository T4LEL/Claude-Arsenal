# Pitch

Fundraising and presentation materials that survive due diligence. Every number is sourced from real data or explicitly labeled as an assumption — no exceptions.

Copy this checklist and check off items as you complete them:

```
Pitch Progress:
- [ ] Step 1: Narrative
- [ ] Step 2: Numbers
- [ ] Step 3: Materials
- [ ] Step 4: Objection prep
- [ ] Step 5: Honesty gate (mandatory)
- [ ] Report
```

## Step 1 — Narrative

Delegate to the **biz-strategist** agent with the product and market context: the problem, why now, the wedge, moat, and an honest read of the competition. Weak or hand-wavy answers here get flagged, not smoothed over — a shaky narrative should read as shaky in the deck, not be papered over with adjectives.

For the market slide, delegate to the **market-analyst** agent: TAM/SAM/SOM sized bottom-up with sources and the arithmetic shown — investors check that math first.

## Step 2 — Numbers

Delegate to the **finance-analyst** agent with the real data available: traction to date (real numbers only — never invented), unit economics, and projections. Every projection row gets its assumption labeled inline (growth rate, conversion, churn) so a reader can see exactly what's fact and what's forecast. Also nail down the ask (amount, instrument) and the use of funds.

## Step 3 — Materials

Delegate to the **copywriter** agent with the narrative and numbers: a 10-12 slide deck outline, one message per slide, each headline written as the takeaway sentence (not a topic label). Also produce a one-pager. If final formatted documents are needed, hand off to the **docs-writer** agent.

## Step 4 — Objection prep

Work out the top 5 objections an investor will raise (market size, competition, team gaps, moat, timing) and write honest answers. Anything genuinely unanswerable becomes its own risks slide in the deck — never an omission the investor discovers in diligence.

## Step 5 — Honesty gate (mandatory)

Scan every number and claim produced across Steps 1-4 — narrative (market size), numbers, deck/one-pager, and objection answers — no exceptions. Each one is either sourced (cite where — traction dashboard, contract, real usage log) or explicitly labeled an assumption. Any number recalled from memory without a source is an unverified assumption, labeled as such — never presented as fact. This step blocks completion; do not report done until it's finished.

## Report

- **Materials:** file paths for the deck outline, one-pager, and any docs-writer output.
- **Key numbers table:** traction, unit economics, projections — each row flagged sourced or assumption.
- **The ask:** amount, instrument, use of funds.
- **Top objections & answers:** the 5 from Step 4.
- **Assumptions register:** every labeled assumption in one place, so the user knows exactly what to defend or validate before the next meeting.
